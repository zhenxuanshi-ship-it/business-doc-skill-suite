# Output Patterns

## Pattern 1: Executive summary first

Start with:
- overall verdict
- strongest strength
- biggest blocker
- minimum changes needed

## Pattern 2: Issue format

For each issue, include:
- title
- severity
- location
- why it matters
- suggested fix

## Pattern 3: Revision roadmap

Group issues into:

### Fix now
Critical and High items that block approval or implementation

### Fix next
Important Medium items that improve clarity and reduce downstream rework

### Optional improvements
Low-severity polish and readability improvements

## Pattern 4: Final recommendation labels

Use one label:
- Ready to use
- Usable after minor revision
- Needs major revision
- Not decision-ready
