# Project Proposal Guide

## Default structure

1. Background
2. Objectives
3. Project scope
4. Main work items
5. Expected outcomes
6. Implementation path
7. Resource / budget estimate
8. Risks and safeguards
9. Recommendation

## What makes a good proposal

- approval-oriented, not academically exhaustive
- clear about value, scope, and expected result
- easy for a decision-maker to scan
- does not bury the ask

## Good patterns

- state the problem in plain language
- define success in concrete terms
- keep scope controlled
- make the requested decision explicit

## Bad patterns

- lots of background, no clear request
- inflated language, little operational detail
- objectives not measurable
- expected results disconnected from actual work items
