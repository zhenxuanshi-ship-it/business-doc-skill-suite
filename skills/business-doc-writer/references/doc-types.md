# Document Type Selection Guide

Use this guide to map the user's request to the right output.

## feasibility-report

Use when the user asks:
- 写可行性研究报告
- 写可研
- 论证这个项目值不值得做
- compare options before approval
- 需要做可行性分析

Best when the key question is:
- should we do this?
- is it feasible?
- which option is better?

## project-proposal

Use when the user asks:
- 写项目建议书
- 写立项材料
- 做一份给领导/甲方的项目提案
- summarize a project for approval

Best when the key question is:
- what is this project?
- why approve it?
- what results will it bring?

## prd

Use when the user asks:
- 写 PRD
- 写产品需求文档
- 整理需求
- 功能需求说明书

Best when the key question is:
- what exactly should be built?
- for whom?
- how do we know it is done?

## architecture-design

Use when the user asks:
- 写架构设计
- 写技术方案
- 写系统设计
- 输出模块设计和数据流

Best when the key question is:
- how should this be built technically?
- how do modules fit together?
- what trade-offs are involved?

## implementation-plan

Use when the user asks:
- 写实施方案
- 做 rollout plan
- 拆阶段和里程碑
- 安排资源与交付计划

Best when the key question is:
- how do we execute this?
- in what order?
- with what resources and risks?

## executive-summary

Use when the user asks:
- 帮我写一页摘要
- 给领导一页版
- 写决策摘要
- 写管理层版本

Best when the key question is:
- what does the decision-maker need to know quickly?
