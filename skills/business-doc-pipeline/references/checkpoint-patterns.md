# Checkpoint Patterns

## Full checkpoint

Use when the user is at an important transition.

```md
## Pipeline checkpoint
- Current stage: draft
- Document type: prd
- Current status: first draft can be generated
- Main gap: acceptance criteria and non-functional constraints are still thin
- Recommended next step: draft the PRD first, then run a product+technical review
```

## Light checkpoint

Use when the next action is obvious.

```md
Current stage is `review`. You already have a draft. Next best step: run a full review and produce a revision roadmap.
```

## Revision checkpoint

Use after review.

```md
## Review checkpoint
- Verdict: Needs major revision
- Fix now: scope boundary, acceptance criteria, risk section
- Next step: revise the draft before finalization
```
