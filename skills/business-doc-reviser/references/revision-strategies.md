# Revision Strategies

## Surgical

Use when:
- the structure is basically sound
- only a few local issues need fixing
- wording, precision, or explicitness is the main problem

Typical actions:
- clarify a sentence or paragraph
- add missing acceptance criteria
- tighten a conclusion
- reconcile a local inconsistency

## Sectional

Use when:
- one or two sections are weak enough that patching is messy
- the rest of the document is mostly stable

Typical actions:
- rewrite risk section
- rewrite recommendation section
- rewrite architecture overview
- rewrite scope and non-scope section

## Structural

Use when:
- the document flow itself is broken
- sections mix multiple concerns
- recommendation, scope, and implementation logic are out of order

Typical actions:
- reorder sections
- split mixed sections
- rebuild argument flow
- narrow scope before rewriting details
