# Question Strategy

Ask fewer questions, but make them directional.

## Ask only if the answer changes the next document or scope

Good examples:
- Who is the primary audience for the next document?
- Is the immediate goal approval, alignment, or implementation?
- Do you want to define scope first, or go directly to a proposal/PRD?

## Avoid asking for everything up front

Bad examples:
- asking for every feature detail before producing a brief
- asking for exact budget when the current task is only discovery
- asking technical implementation details before goals are stable

## Preferred sequence

1. audience
2. objective
3. scope boundary
4. strongest constraint

That is usually enough to produce a useful discovery artifact.
