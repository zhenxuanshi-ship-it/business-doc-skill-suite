# Input Patterns

## Meeting notes

Typical signals:
- lots of bullets
- multiple speakers' opinions mixed together
- decisions and open issues mixed together

Best mode:
- `notes-to-brief`

Extract:
- decisions already made
- unresolved issues
- implied owners
- blocked items

## Rough idea

Typical signals:
- “我想做一个…”
- “我们考虑搞一个…”
- no clear audience or scope yet

Best mode:
- `brief`

Extract:
- problem statement
- possible target audience
- why this matters
- missing basics

## Scope confusion

Typical signals:
- many feature ideas mixed together
- unclear boundary between phase 1 and future scope
- implementation and goals mixed together

Best mode:
- `scope-clarification`

Extract:
- in-scope
- out-of-scope
- assumptions
- dependencies

## Requirement fragments

Typical signals:
- feature bullets
- process demands
- user complaints
- no formal PRD yet

Best mode:
- `requirements-seed`

Extract:
- user roles
- scenarios
- requirement seeds
- NFR seeds
- open product questions

## Approval-oriented input

Typical signals:
- leadership instruction
- need to justify project launch
- focus on necessity, value, and approval

Best mode:
- `proposal-intake`

Extract:
- why now
- expected outcome
- business value
- risk prompts
- suggested next doc
