# Next Document Mapping

## Choose `project-proposal` when
- approval is the next goal
- leadership or sponsor needs a formal proposal
- the question is whether to initiate the project

## Choose `feasibility-report` when
- the key issue is whether the project is worth doing
- options need comparison
- cost-benefit / feasibility needs explicit analysis

## Choose `prd` when
- the product direction is accepted
- requirements need clarification
- product / design / engineering need a build-ready document

## Choose `architecture-design` when
- requirements mostly exist
- the next question is how to build it technically
- module boundaries and technical choices need definition

## Choose `implementation-plan` when
- project scope is mostly known
- the next issue is sequencing, rollout, owners, and milestones
