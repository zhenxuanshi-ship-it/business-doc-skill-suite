# Business Doc Suite

商务文档智能生产套件主入口

这是 Business Doc Suite 的总入口 skill。当你需要生产一份完整的商务文档时，使用这个 skill 进行流程编排。

## 核心功能

这是一个**流程编排型 skill**，不是直接写正文。它的职责是：

1. **识别用户需求**：判断用户要写什么类型的文档
2. **规划写作流程**：确定需要走哪些阶段（Discovery → Writer → Review → Integrity → Reviser）
3. **逐步执行**：按顺序调用各个阶段
4. **产出最终文档**：生成可用于交付的文档

## 使用场景

**适用场景**：
- "按照写作技能套件写一个智慧园区项目建议书"
- "用完整流程写一份可研报告"
- "走一套文档生产流程"
- "用biz-doc套件帮我写一个方案"
- "start business doc workflow"

**不适用场景**：
- 已经明确知道写什么，只需要单独调用某个 skill 时（如"帮我 review 这份文档"）

## 流程模式

### 完整流程（Full Pipeline）

适用于：全新文档、从零开始、需要高质量输出

```
Discovery → Writer → Review → Integrity → Reviser
```

### 快速流程（Quick Pipeline）

适用于：已有初稿、需要快速提升

```
Review → Integrity → Reviser
```

### 写作流程（Writing Pipeline）

适用于：只需要写作，不需要前面 Discovery 环节

```
Writer
```

## 执行步骤

### Step 1: 需求识别

首先判断用户需求：

1. **文档类型**：项目建议书、可研报告、PRD、架构设计、实施方案等
2. **输入状态**：是完全从零开始，还是已有初稿
3. **流程需求**：要完整流程，还是只需要某个环节

### Step 2: 流程规划

根据需求规划流程：

| 用户情况 | 推荐流程 |
|---|---|
| 零基础，只有想法 | 完整流程 |
| 有brief/需求文档 | Writer → Review → Integrity → Reviser |
| 已有初稿 | Review → Integrity → Reviser |
| 只要快速起草 | Writer |

### Step 3: 阶段执行

按顺序执行各个阶段，每个阶段完成后：
- 总结本阶段产出
- 询问用户是否继续
- 根据反馈决定下一步

### Step 4: 最终交付

完成所有阶段后：
- 整理最终文档
- 告知用户文档位置
- 询问是否需要调整

## 输出格式

每个阶段执行后，输出格式如下：

```
---

## [阶段名称] 完成

### 产出
- [产出物描述]

### 下一步
- [建议的下一步操作]
- 用户确认后继续

---
```

## 关键规则

1. **不跳过阶段**：除非用户明确要求，否则按流程顺序执行
2. **每步确认**：每个阶段完成后，询问用户是否继续
3. **记录上下文**：记录已完成的阶段和产出，避免重复
4. **灵活应变**：根据用户反馈调整流程

## 触发示例

| 用户输入 | 识别结果 | 执行流程 |
|---|---|---|
| 写一个项目建议书 | 项目建议书 + 零基础 | 完整流程 |
| 帮我写一份可研报告 | 可行性研究报告 + 零基础 | 完整流程 |
| 这份PRD帮我review一下 | PRD + 已有初稿 | Review → Integrity → Reviser |
| 用套件写一个方案 | 未知类型 + 需确认 | 先询问文档类型 |

## 与其他Skill的关系

- **biz-discovery**：需求梳理阶段调用
- **business-doc-writer**：写作阶段调用
- **business-doc-reviewer**：评审阶段调用
- **business-doc-integrity**：校验阶段调用
- **business-doc-reviser**：修订阶段调用

本skill是**编排者**，负责调度这些skill，而不是自己完成具体的写作/评审工作。

## 注意事项

1. 如果用户只是要某个单独的skill（如"帮我review"），不要触发本skill，直接调用对应的skill
2. 如果用户需求不明确，先询问确认后再规划流程
3. 每个阶段都要等用户确认后再继续下一步
4. 最终文档要保存到用户可访问的位置

## 启动引导语

当需要确认流程时，可以使用以下引导语：

```
我理解你要写一份[文档类型]。

推荐流程：[完整流程/快速流程]

包含以下阶段：
1. 需求梳理（Discovery）
2. 文档写作（Writer）
3. 文档评审（Review）
4. 一致性校验（Integrity）
5. 修订定稿（Reviser）

是否按此流程执行？
```

或者：

```
你提到要写[文档类型]，但我需要确认一下：

1. 是从零开始，还是已有初稿？
2. 需要完整流程还是只需要某些环节？

请告诉我你的需求，我来安排合适的流程。
```
